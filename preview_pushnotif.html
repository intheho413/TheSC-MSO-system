⑤ 동작 흐름 (A안: PWA 클라이언트)

1

다른 사용자가 변경

이대리가 안건에 코멘트 입력 → Supabase DB UPDATE

2

Realtime 푸시

Supabase Realtime이 모든 구독 클라이언트에 변경 이벤트 전달 (수 ms)

3

알림 조건 검사

내 안건? 작성자 ≠ 본인? 알림 종류 ON? 페이지 비활성? → 통과 시 표시

4

OS 알림센터에 표시

Service Worker가 OS 네이티브 알림 호출 → 잠금화면/배너/뱃지

A안의 특징: 별도 푸시 서버 없이 Supabase Realtime 구독만으로 동작합니다. 장점: 추가 비용 0원, 구현 단순, 즉시 사용 가능. 제약: 브라우저/PWA가 실행 중일 때만 받습니다 (백그라운드 탭 OK / 완전 종료 시 X). 실무에선 보통 충분 — 영업 담당자는 앱을 자주 켜둠. 완전 백그라운드 푸시가 필요해지면 B(OneSignal) 또는 C(Edge Function)로 업그레이드.
